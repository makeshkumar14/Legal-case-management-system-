## 1. Project Overview

**Project Name:** Legal Case Management System  
**Architecture:** Full-Stack Web Application  
**Frontend:** React 19 + Vite 7 (JSX, TailwindCSS 4)  
**Backend:** Python Flask 3.1 REST API  
**Database:** MySQL (via PyMySQL + SQLAlchemy ORM)  
**Authentication:** JWT (JSON Web Tokens)  
**Version Control:** Git (GitHub â€” `makeshkumar14/Legal-case-management-system-`)

### Purpose

A role-based legal case management system enabling **Public Users**, **Advocates**, and **Court Officials** to manage cases, hearings, documents, tasks, messaging, and analytics through a unified web platform.

## 6. Frontend Architecture

### Routing & Role-Based Access Control (RBAC)

The app implements three distinct user portals with **protected routes**:

| Role                  | Base Path   | Pages                                                                                                           |
| --------------------- | ----------- | --------------------------------------------------------------------------------------------------------------- |
| **Public** (Citizen)  | `/public`   | Dashboard, Cases, Case Detail, Search, Notifications, Messages, Settings                                        |
| **Advocate** (Lawyer) | `/advocate` | Dashboard, Cases, Case Detail, Evidence, Calendar, Notes, Tasks, Performance, Messages, Settings, Notifications |
| **Court** (Official)  | `/court`    | Dashboard, Cases, Case Detail, Hearings, Advocates, Analytics, QR, Rooms, Messages, Settings, Notifications     |

- Unauthenticated users are redirected to `/login`
- Users accessing wrong role routes are redirected to their own portal
- JWT tokens are stored in `localStorage` and attached to all API requests via Axios interceptor

### State Management

- **AuthContext**: Manages user authentication state (user object, JWT token, login/logout functions)
- **ThemeContext**: Theme management (currently locked to light mode)
- **ToastProvider**: Global toast notification system

### Key UI Components (12 Shared Components)

1. **Navbar** â€” Top bar with search, notification bell, user avatar, theme toggle
2. **Sidebar** â€” Role-specific navigation with color themes (purple for public, green for advocate, blue for court)
3. **ChatBot** â€” AI assistant chatbot interface
4. **Modal** â€” Reusable dialog component
5. **Toast** â€” Context-based toast notifications
6. **AnimatedCard** â€” Framer Motion card wrapper
7. **StatusBadge** â€” Color-coded case status indicators
8. **Timeline** â€” Case timeline visualization
9. **LoadingSkeleton** â€” Skeleton loading placeholders
10. **CountdownTimer** â€” Countdown to next hearing
11. **QRCodeScanner** â€” Scan QR codes for case lookup
12. **QRCodeViewer** â€” Display QR codes for cases
